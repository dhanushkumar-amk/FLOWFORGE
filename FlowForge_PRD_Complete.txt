1. PRODUCT OVERVIEW
────────────────────────────────────────────────────────────────────────────────

FlowForge is a production-grade, real-time collaborative platform that lets
teams visually design, collaborate on, and automatically execute complex
workflows modeled as Directed Acyclic Graphs (DAGs).

PROBLEM IT SOLVES:
  - Manual tracking of dependent tasks
  - Unreliable background execution
  - Lack of live collaboration in tools like Linear, ClickUp, or n8n

CORE VALUE PROPOSITION:
  Build once → Collaborate live → Execute reliably at scale → Monitor everything.

TARGET USERS:
  - Product teams
  - Agencies
  - Startups
  - Freelancers

MONETIZATION:
  Freemium model — Free tier + Paid workspaces via Razorpay subscriptions

8. DSA IMPLEMENTED & EXPLAINED
────────────────────────────────────────────────────────────────────────────────

All DSA implementations will be documented in code + README with complexity
analysis and real-world justification.

  1. TOPOLOGICAL SORT
     ─────────────────
     Algorithms : Kahn's Algorithm + DFS-based
     Purpose    : Determines correct DAG execution order
     Complexity : O(V + E) where V = nodes, E = edges

  2. DFS CYCLE DETECTION
     ────────────────────
     Algorithm  : Depth-First Search with visited/recursion-stack tracking
     Purpose    : Prevents invalid workflows (ensures graph is truly a DAG)
     Complexity : O(V + E)

  3. PRIORITY QUEUE (BINARY HEAP)
     ─────────────────────────────
     Data Structure : Min-Heap / Max-Heap
     Purpose        : Scheduled & priority-based task execution
     Complexity     : O(log N) insert/delete

  4. TOKEN BUCKET ALGORITHM
     ────────────────────────
     Purpose    : Rate limiting implementation (per user/workspace)
     Mechanism  : Fixed bucket capacity refills at constant rate
     Complexity : O(1) per request check
